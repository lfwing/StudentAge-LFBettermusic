LFBetterAudio 公开 Mod 作者模板（1.0.0）

把整个 BetterAudio 文件夹放到你的 Mod 根目录：

<你的 Mod 根目录>/
└─ BetterAudio/
   ├─ Audio/
   ├─ Timeline/
   └─ BetterAudio.json

BetterAudio.json 示例字段：
- audios：音频条目数组
- id：音频 ID
- name：显示名称
- audioPath：Audio 目录中的音频相对路径；留空时尝试使用同 ID 的原版音频
- timelinePath：Timeline 目录中的 .lrc 相对路径；可留空
- volume：播放音量
- type：1=音乐，2=音效；缺省或非法值按 1

Timeline 当前仍使用原有 .lrc 解析和显示逻辑，本版本只完成命名调整，不增加新事件语法。

主要 EFFECT：
- 1163,0：停止音乐和全部插件音效并刷新通道。
- 1163,1,x,y,z,m[,u[,v]]：单 Talk 背景型音乐。
- 1163,2,x,y,z,id1,id2...,-1[,u[,v]]：单 Talk 唱歌型音乐。
- 1163,3,x,y[,u[,v]]：单 Talk 音效，y=1 单次、y=2 循环。
- 1163,10,x,y,z,m[,u[,v]]：跨 Talk 背景型音乐；y=0 无限循环，y>0 为总播放次数。
- 1163,20,x,y,z,id1,id2...,-1[,u[,v]]：跨 Talk 唱歌型音乐。
- 1163,30,x,y[,u[,v]]：跨 Talk 音效，y=1 单次、y=2 循环。
- 1163,99,1 / 2：暂停 / 恢复当前音乐。
- 1163,99,3 / 4：暂停 / 恢复最近建立且仍有效的插件音效。
- 1163,99,10 / 20：音乐淡出暂停 / 淡入恢复。

类型规则：
- 1163,1/2/10/20 仅接受 type=1 的音乐。
- 1163,3/30 仅接受 type=2 的音效。
- 音乐取得原版 BGM 通道优先控制；音效不抢占，可与音乐及其他插件音效共存。
- 同一 Talk 先执行播放型指令，再执行暂停、恢复、停止等控制型指令。
- 加载期间收到暂停指令时使用 PendingPauseAfterLoad，加载完成后直接暂停，不先发声。

u/v：
- u=0 从音频开头开始；u>=1 从第 u 条有效 Timeline 时间记录开始。
- v 为包含式终点，实际停止点取第 v 条之后的下一条有效时间记录。
- 音效不显示 Timeline 文本，但 1163,3/30 可把 .lrc 时间标签用于播放区间。
- 填写 u/v 时必须存在有效 Timeline；不填写时可不配置 timelinePath。

32 色背景歌词颜色：
0白色；
1正红，2朱红，3深红，4酒红；
5珊瑚粉，6桃粉，7樱花粉，8玫瑰粉；
9橙红，10橙色，11杏橙；
12金黄，13明黄，14柠檬黄，15米黄；
16黄绿，17青柠绿，18草绿，19翠绿，20薄荷绿；
21青绿，22青色，23天蓝，24湖蓝，25宝蓝，26靛蓝；
27蓝紫，28淡紫，29紫色，30紫红，31洋红。
非法颜色值按 0（白色）处理。

歌词位置锁：
- 工具栏为设置、复位、位置锁。
- 锁定后位置跨新播放指令保留，禁止拖动，灰框保持显示。
- 解锁后恢复拖动和自动隐藏；字号与颜色仍按新指令初始化。

重要：
1. 相对路径以当前 Mod 自己的 BetterAudio 目录为基准。
2. 1163001 是代码级保留校验音频 ID。
3. 不同 Mod 应避免使用重复音频 ID。
4. 上传创意工坊后，插件直接扫描各 Mod 根目录下的 BetterAudio 资源包。
